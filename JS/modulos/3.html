03

Funciones.

En este módulo, exploraremos los conceptos básicos de JavaScript, incluyendo funciones, parámetros y retorno de valores.

Contenido del Modulo:

Definición y uso de funciones

Las funciones son bloques de código reutilizables que realizan una tarea específica. Se definen utilizando la palabra clave function y pueden aceptar parámetros y devolver valores.

Parámetros y valores de retorno

Las funciones pueden aceptar parámetros, que son valores que se pasan a la función cuando se llama. Además, las funciones pueden devolver valores utilizando la palabra clave return.

Funciones flecha (=>)

Las funciones flecha son una sintaxis más concisa para definir funciones en JavaScript. Utilizan la flecha => y no tienen su propio contexto this.

Funciones recursivas

Las funciones recursivas son aquellas que se llaman a sí mismas para resolver un problema. Son útiles para tareas que pueden dividirse en subproblemas más pequeños.

Funciones de orden superior

Las funciones de orden superior son aquellas que pueden aceptar otras funciones como argumentos o devolver funciones como resultados. Son fundamentales en la programación funcional.

Funciones anónimas

Las funciones anónimas son funciones que no tienen un nombre. Se utilizan comúnmente como argumentos para otras funciones o como funciones de devolución de llamada.

Ámbito (scope) y contexto (this)

El ámbito (scope) se refiere a la accesibilidad de las variables y funciones en diferentes partes del código. El contexto (this) se refiere al objeto al que pertenece una función cuando se invoca.

Closures

Un closure es una función que recuerda el entorno en el que fue creada, incluso después de que ese entorno haya dejado de existir. Esto permite el acceso a variables externas desde una función interna.

Funciones puras e impuras

Las funciones puras son aquellas que, dado el mismo conjunto de entradas, siempre devuelven el mismo resultado y no tienen efectos secundarios. Las funciones impuras pueden tener efectos secundarios y su salida puede variar.

Definición de funciones

Las funciones en JavaScript se definen utilizando la palabra clave function, seguida del nombre de la función, una lista de parámetros entre paréntesis y un bloque de código entre llaves. Aquí hay un ejemplo básico:

function saludar(nombre) {
    console.log("Hola, " + nombre);
}

En este ejemplo, hemos definido una función llamada saludar que acepta un parámetro llamado nombre. Cuando se llama a esta función, imprimirá un saludo en la consola.

Para llamar a una función, simplemente usamos su nombre seguido de paréntesis, pasando los argumentos necesarios:

saludar("Juan"); // Salida: Hola, Juan

Parámetros y valores de retorno

Las funciones pueden aceptar múltiples parámetros, separados por comas. Aquí hay un ejemplo de una función que suma dos números:

function sumar(a, b) {
    return a + b;
}

En este caso, la función sumar acepta dos parámetros, a y b, y devuelve su suma utilizando la palabra clave return.

Para obtener el valor devuelto por una función, podemos asignarlo a una variable:

let resultado = sumar(5, 3);  // Resultado ahora es 8

Funciones flecha (=>)

Las funciones flecha son una forma más concisa de definir funciones en JavaScript. Utilizan la sintaxis => y no tienen su propio contexto this. Aquí hay un ejemplo de una función flecha que suma dos números:

var a = 5;
var b = 3;
const sumar = (a, b) => {
    return a + b;
};

Si la función tiene una sola expresión, podemos omitir las llaves y la palabra clave return:

const sumar = (a, b) => a + b; // Resultado: 8

Las funciones flecha son especialmente útiles cuando se trabaja con funciones de orden superior, como map, filter y reduce.

Funciones recursivas

Las funciones recursivas son aquellas que se llaman a sí mismas para resolver un problema. Son útiles para tareas que pueden dividirse en subproblemas más pequeños. Aquí hay un ejemplo de una función recursiva que calcula el factorial de un número:

function factorial(n) {
    if (n === 0) {
        return 1;
    } else {
        return n * factorial(n - 1);  // Llamada recursiva secuencial 5*4*3*2*1=120
    }
}
const resultado = factorial(5);  // Resultado: 120

En este ejemplo, la función factorial se llama a sí misma con un valor decreciente hasta que alcanza el caso base de n === 0.

Las funciones recursivas pueden ser más difíciles de entender y depurar que las funciones iterativas, pero son una herramienta poderosa en la programación.

Funciones de orden superior

Las funciones de orden superior son aquellas que pueden aceptar otras funciones como argumentos o devolver funciones como resultados. Son fundamentales en la programación funcional y permiten un alto grado de abstracción y reutilización del código.

Un ejemplo común de una función de orden superior es el método map en los arrays, que aplica una función a cada elemento del array y devuelve un nuevo array con los resultados:

const numeros = [1, 2, 3, 4, 5];
const dobles = numeros.map(n => n * 2);  // Resultado: [2, 4, 6, 8, 10]

En este ejemplo, map es una función de orden superior que toma una función flecha como argumento para duplicar cada número en el array.

Las funciones de orden superior son una herramienta poderosa en la programación funcional y permiten un alto grado de abstracción y reutilización del código.

Funciones anónimas

Las funciones anónimas son funciones que no tienen un nombre. Se utilizan comúnmente como argumentos para otras funciones o como funciones de devolución de llamada (callbacks). Aquí hay un ejemplo de una función anónima utilizada como argumento para el método setTimeout:

setTimeout(function() {
    console.log("Hola, mundo!");
}, 1000);

En este ejemplo, hemos pasado una función anónima como el primer argumento a setTimeout, que se ejecutará después de 1000 milisegundos (1 segundo).

Las funciones anónimas también se pueden definir utilizando la sintaxis de funciones flecha:

setTimeout(() => {
    console.log("Hola, mundo!");
}, 1000);

En este ejemplo, hemos pasado una función anónima como el primer argumento a setTimeout, que se ejecutará después de 1000 milisegundos (1 segundo).

Ámbito (scope) y contexto (this)

El ámbito (scope) se refiere a la accesibilidad de las variables y funciones en diferentes partes del código. JavaScript tiene dos tipos principales de ámbito: global y local. Las variables definidas fuera de cualquier función tienen ámbito global, mientras que las variables definidas dentro de una función tienen ámbito local.

El contexto (this) se refiere al objeto al que pertenece una función cuando se invoca. El valor de this puede variar dependiendo de cómo se llame a la función. En las funciones regulares, this generalmente se refiere al objeto global (o undefined en modo estricto), mientras que en los métodos de un objeto, this se refiere al objeto propietario.

Las funciones flecha no tienen su propio contexto this; en su lugar, heredan el valor de this del entorno en el que fueron creadas.

Closures

Un closure es una función que recuerda el entorno en el que fue creada, incluso después de que ese entorno haya dejado de existir. Esto permite el acceso a variables externas desde una función interna. Aquí hay un ejemplo de un closure:

function crearContador() {
    let contador = 0;
    return function() {
        contador++;
        console.log(contador);
    };
}

const contador1 = crearContador();
contador1(); // 1
contador1(); // 2

const contador2 = crearContador();
contador2(); // 1
contador2(); // 2

En este ejemplo, la función interna devuelta por crearContador es un closure que recuerda la variable contador del entorno externo. Cada vez que llamamos a contador1 o contador2, se incrementa y se imprime su propio contador independiente.

Funciones puras e impuras

Las funciones puras son aquellas que, dado el mismo conjunto de entradas, siempre devuelven el mismo resultado y no tienen efectos secundarios. Esto significa que no modifican ninguna variable externa ni interactúan con el mundo exterior (como la consola o el DOM). Aquí hay un ejemplo de una función pura:

function sumar(a, b) {
    return a + b;
}

En este caso, la función sumar es pura porque siempre devuelve la misma salida para las mismas entradas y no tiene efectos secundarios.
