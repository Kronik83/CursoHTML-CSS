Etiqueta <img>

La etiqueta <img> es fundamental para mostrar imágenes en páginas web. Es una etiqueta auto-cerrada que requiere atributos específicos para funcionar correctamente.

Atributos esenciales

src (source)

Especifica la ruta o URL de la imagen que se va a mostrar:

Ejemplos de src:

<!-- Imagen local -->
<img src="imagenes/foto.jpg">

<!-- Imagen desde URL -->
<img src="https://ejemplo.com/imagen.png">

<!-- Ruta relativa -->
<img src="../assets/logo.svg">

alt (alternative text)

Proporciona texto alternativo para accesibilidad y SEO. Es obligatorio y crucial:

  • Se muestra si la imagen no carga
  • Leído por lectores de pantalla
  • Mejora el SEO de la página
  • Describe el contenido de la imagen

Ejemplos de alt:

<img src="perro.jpg" alt="Perro golden retriever jugando en el parque">
<img src="logo.png" alt="Logo de la empresa TechCorp">
<img src="grafico.png" alt="Gráfico de ventas enero-marzo 2025">

width y height

Controlan las dimensiones de la imagen en píxeles:

Controlando dimensiones:

<!-- Tamaño específico -->
<img src="imagen.jpg" alt="Descripción" width="300" height="200">

<!-- Solo ancho (altura proporcional) -->
<img src="imagen.jpg" alt="Descripción" width="500">

<!-- Mejor práctica: usar CSS -->
<img src="imagen.jpg" alt="Descripción" class="imagen-responsive">

Otros atributos útiles

  • title: Tooltip que aparece al pasar el mouse
  • loading="lazy": Carga diferida para mejor rendimiento
  • crossorigin: Control de CORS para imágenes externas
  • decoding="async": Decodificación asíncrona

Ejemplo completo:

<img 
  src="imagenes/producto.jpg" 
  alt="Laptop gaming con RGB y pantalla 4K" 
  width="400" 
  height="300"
  title="Ver detalles del producto"
  loading="lazy">

Formatos de imagen recomendados

  • JPEG: Fotos y imágenes con muchos colores
  • PNG: Imágenes con transparencia, logos
  • SVG: Gráficos vectoriales, iconos
  • WebP: Formato moderno, mejor compresión
  • AVIF: Formato de nueva generación

Mejores prácticas

  • Siempre incluye el atributo alt
  • Optimiza el tamaño de las imágenes
  • Usa formatos modernos cuando sea posible
  • Considera la carga diferida (lazy loading)
  • Proporciona múltiples tamaños para responsive design

Video y audio en HTML

HTML5 introdujo las etiquetas <video> y <audio> para incrustar contenido multimedia directamente en las páginas web sin necesidad de plugins externos.

Etiqueta <video>

Permite incorporar videos de forma nativa en HTML:

Ejemplo básico de video:

<video controls width="600" height="400">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Tu navegador no soporta el elemento video.
</video>

Atributos de video

  • controls: Muestra controles de reproducción
  • autoplay: Reproduce automáticamente (úsalo con cuidado)
  • loop: Reproduce en bucle
  • muted: Silencia el audio por defecto
  • poster: Imagen que se muestra antes de reproducir
  • preload: Controla la precarga (none, metadata, auto)

Video con todos los atributos:

<video 
  controls 
  width="800" 
  height="450"
  poster="miniatura.jpg"
  preload="metadata">
  
  <source src="video-hd.mp4" type="video/mp4">
  <source src="video-hd.webm" type="video/webm">
  
  <!-- Subtítulos -->
  <track src="subtitulos-es.vtt" kind="subtitles" srclang="es" label="Español">
  
  <p>Tu navegador no soporta video HTML5. 
     <a href="video-hd.mp4">Descargar video</a>
  </p>
</video>

Etiqueta <audio>

Similar a video, pero para contenido de audio:

Ejemplo de audio:

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  <source src="audio.wav" type="audio/wav">
  Tu navegador no soporta el elemento audio.
</audio>

Atributos de audio

  • controls: Muestra controles de reproducción
  • autoplay: Reproduce automáticamente
  • loop: Reproduce en bucle
  • muted: Silencia por defecto
  • preload: Controla la precarga

Formatos multimedia recomendados

Video:

  • MP4 (H.264): Amplia compatibilidad
  • WebM (VP9): Código abierto, buena compresión
  • AV1: Formato de nueva generación

Audio:

  • MP3: Amplia compatibilidad
  • OGG Vorbis: Código abierto
  • AAC: Buena calidad y compresión
  • FLAC: Sin pérdida de calidad

Mejores prácticas multimedia

  • Proporciona múltiples formatos para compatibilidad
  • Incluye texto alternativo para accesibilidad
  • Optimiza el tamaño de los archivos
  • Considera el autoplay (puede ser molesto)
  • Usa poster para videos
  • Proporciona controles siempre

Ejemplo de audio con playlist:

<div class="playlist">
  <h3>Mi Playlist</h3>
  
  <audio controls>
    <source src="cancion1.mp3" type="audio/mpeg">
    <source src="cancion1.ogg" type="audio/ogg">
  </audio>
  <p>Canción 1 - Artista A</p>
  
  <audio controls>
    <source src="cancion2.mp3" type="audio/mpeg">
    <source src="cancion2.ogg" type="audio/ogg">
  </audio>
  <p>Canción 2 - Artista B</p>
</div>

Figure y figcaption

Los elementos <figure> y <figcaption> proporcionan una forma semánticamente correcta de agrupar contenido multimedia con sus descripciones o títulos.

Elemento <figure>

Se usa para agrupar contenido que es referenciado desde el contenido principal pero que puede moverse a otra ubicación sin afectar el flujo del documento:

  • Imágenes con descripción
  • Diagramas y gráficos
  • Fragmentos de código
  • Videos con título
  • Citas con atribución

Elemento <figcaption>

Proporciona una descripción o título para el contenido dentro de <figure>. Debe ser el primer o último elemento hijo de figure.

Ejemplo básico con imagen:

<figure>
  <img src="paisaje.jpg" alt="Paisaje montañoso al atardecer">
  <figcaption>
    Atardecer en los Andes peruanos - Fotografía de Juan Pérez
  </figcaption>
</figure>

Casos de uso comunes

1. Galería de imágenes

<figure>
  <img src="producto1.jpg" alt="Smartphone modelo X">
  <figcaption>
    <strong>Smartphone X</strong> - Pantalla 6.1", 128GB, $599
  </figcaption>
</figure>

<figure>
  <img src="producto2.jpg" alt="Laptop gaming RGB">
  <figcaption>
    <strong>Gaming Laptop Pro</strong> - RTX 4060, 16GB RAM, $1299
  </figcaption>
</figure>

2. Video con descripción

<figure>
  <video controls width="600" poster="tutorial-thumb.jpg">
    <source src="tutorial-html.mp4" type="video/mp4">
    <source src="tutorial-html.webm" type="video/webm">
  </video>
  <figcaption>
    Tutorial: Primeros pasos con HTML - Duración: 15 minutos
  </figcaption>
</figure>

3. Fragmento de código

<figure>
  <pre><code>
function saludar(nombre) {
  return `¡Hola, ${nombre}!`;
}
  </code></pre>
  <figcaption>
    Función JavaScript para saludar usuarios
  </figcaption>
</figure>

4. Gráfico con datos

<figure>
  <img src="grafico-ventas.svg" alt="Gráfico de barras de ventas 2025">
  <figcaption>
    <strong>Figura 1:</strong> Evolución de ventas por trimestre en 2025. 
    Se observa un crecimiento del 25% respecto al año anterior.
  </figcaption>
</figure>

5. Audio con información

<figure>
  <audio controls>
    <source src="podcast-ep1.mp3" type="audio/mpeg">
    <source src="podcast-ep1.ogg" type="audio/ogg">
  </audio>
  <figcaption>
    <strong>Podcast TechTalk - Episodio 1:</strong> 
    "El futuro del desarrollo web" con María González
  </figcaption>
</figure>

Ventajas semánticas

  • Accesibilidad: Los lectores de pantalla entienden la relación
  • SEO: Los motores de búsqueda comprenden mejor el contenido
  • Estructura: Código más limpio y organizad
  • Estilizado: Más fácil de estilizar con CSS
  • Mantenimiento: Código más fácil de mantener

Mejores prácticas

  • Usa figure solo cuando el contenido necesite descripción
  • Coloca figcaption al inicio o al final de figure
  • Haz que figcaption sea descriptivo pero conciso
  • No uses figure para decoración, solo para contenido relevante
  • Considera la numeración para figuras en artículos largos

Ejemplo completo - Artículo con múltiples figuras:

<article>
  <h1>Guía de fotografía de paisajes</h1>
  
  <p>La fotografía de paisajes requiere técnica y paciencia...</p>
  
  <figure>
    <img src="equipo-fotografia.jpg" alt="Cámara y trípode en montaña">
    <figcaption>
      <strong>Figura 1:</strong> Equipo básico para fotografía de paisajes
    </figcaption>
  </figure>
  
  <p>Como se puede ver en la Figura 1, el equipo esencial incluye...</p>
  
  <figure>
    <video controls poster="tecnica-composicion.jpg">
      <source src="composicion-regla-tercios.mp4" type="video/mp4">
    </video>
    <figcaption>
      <strong>Video tutorial:</strong> Aplicando la regla de los tercios
    </figcaption>
  </figure>
</article>

Ejercicio Práctico

Galería multimedia personal

Intermedio

Objetivo: Inserta una imagen de perfil y un video de presentación o un audio corto de bienvenida usando las técnicas aprendidas.

Pasos a seguir:

  1. Crea una sección "Sobre mí" con una imagen de perfil usando figure y figcaption
  2. Agrega un video de presentación personal (puedes usar un placeholder)
  3. Incluye un audio de bienvenida con controles
  4. Asegúrate de usar todos los atributos necesarios (alt, controls, etc.)
  5. Aplica las mejores prácticas de accesibilidad

Estructura sugerida:

<section id="sobre-mi">
  <h2>Sobre mí</h2>
  
  <figure>
    <img src="perfil.jpg" alt="Tu nombre - Desarrollador web" width="300">
    <figcaption>Tu Nombre - Desarrollador Frontend</figcaption>
  </figure>
  
  <figure>
    <video controls width="600" poster="video-thumb.jpg">
      <source src="presentacion.mp4" type="video/mp4">
      <source src="presentacion.webm" type="video/webm">
      No se puede cargar el video.
    </video>
    <figcaption>Video de presentación personal</figcaption>
  </figure>
  
  <figure>
    <audio controls>
      <source src="bienvenida.mp3" type="audio/mpeg">
      <source src="bienvenida.ogg" type="audio/ogg">
      No se puede cargar el audio.
    </audio>
    <figcaption>Mensaje de bienvenida</figcaption>
  </figure>
</section>

Puntos extra:

  • Agrega subtítulos al video
  • Usa lazy loading en las imágenes
  • Crea una galería con múltiples imágenes
  • Implementa controles personalizados con CSS
