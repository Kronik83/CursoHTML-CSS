Etiquetas semánticas modernas

HTML5 introdujo elementos semánticos que proporcionan significado al contenido, mejorando la estructura, accesibilidad y SEO de las páginas web. Estos elementos reemplazan el uso genérico de <div> con elementos más descriptivos.

Elemento <header>

Representa el encabezado de una sección o página. Típicamente contiene:

  • Logo del sitio
  • Título principal
  • Navegación principal
  • Información de contacto

Ejemplo de header:

<header>
  <img src="logo.png" alt="Logo de la empresa">
  <h1>Mi Sitio Web</h1>
  <p>Bienvenido a nuestra página oficial</p>
</header>

Elemento <nav>

Define una sección de navegación con enlaces a otras páginas o secciones:

Ejemplo de navegación:

<nav>
  <ul>
    <li><a href="#inicio">Inicio</a></li>
    <li><a href="#servicios">Servicios</a></li>
    <li><a href="#acerca">Acerca de</a></li>
    <li><a href="#contacto">Contacto</a></li>
  </ul>
</nav>

Elemento <main>

Representa el contenido principal del documento. Solo debe haber uno por página:

<main>
  <h1>Contenido principal de la página</h1>
  <p>Este es el contenido más importante...</p>
</main>

Elemento <section>

Define una sección temática del documento. Cada section debería tener un encabezado:

Ejemplo de secciones:

<section>
  <h2>Nuestros Servicios</h2>
  <p>Ofrecemos una amplia gama de servicios...</p>
</section>

<section>
  <h2>Testimonios</h2>
  <p>Lo que dicen nuestros clientes...</p>
</section>

Elemento <article>

Representa contenido independiente que puede ser distribuido por separado:

  • Entradas de blog
  • Artículos de noticias
  • Comentarios de usuarios
  • Productos en un catálogo

Ejemplo de artículo:

<article>
  <header>
    <h2>Introducción a HTML5</h2>
    <p>Publicado el <time datetime="2025-10-21">21 de octubre, 2025</time></p>
  </header>
  
  <p>HTML5 es la última versión del lenguaje de marcado...</p>
  
  <footer>
    <p>Autor: Juan Desarrollador</p>
  </footer>
</article>

Elemento <aside>

Contenido relacionado pero separado del contenido principal:

  • Barras laterales
  • Información adicional
  • Publicidad
  • Enlaces relacionados

Ejemplo de aside:

<aside>
  <h3>Artículos relacionados</h3>
  <ul>
    <li><a href="#">CSS para principiantes</a></li>
    <li><a href="#">JavaScript básico</a></li>
    <li><a href="#">Responsive Design</a></li>
  </ul>
</aside>

Elemento <footer>

Representa el pie de página de una sección o documento:

Ejemplo de footer:

<footer>
  <p>© 2025 Mi Empresa. Todos los derechos reservados.</p>
  <nav>
    <a href="#privacidad">Política de Privacidad</a>
    <a href="#terminos">Términos de Uso</a>
  </nav>
</footer>

Comparación: Antes vs Después

Estructura tradicional (no semántica):

<div id="header">
  <div id="logo">...</div>
  <div id="nav">...</div>
</div>
<div id="content">
  <div class="post">...</div>
  <div class="sidebar">...</div>
</div>
<div id="footer">...</div>

Estructura semántica (HTML5):

<header>
  <img src="logo.png" alt="Logo">
  <nav>...</nav>
</header>
<main>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>

Organización de contenido semánticamente

La organización semántica del contenido no solo mejora la estructura del código, sino que también facilita el mantenimiento, la accesibilidad y el SEO de tu sitio web.

Principios de organización semántica

  • Jerarquía clara: Usa encabezados (h1-h6) de forma lógica
  • Agrupación temática: Usa section para agrupar contenido relacionado
  • Contenido independiente: Usa article para contenido autónomo
  • Información complementaria: Usa aside para contenido secundario

Estructura típica de una página web

Plantilla básica semántica:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi Sitio Web</title>
</head>
<body>
  <header>
    <h1>Nombre del Sitio</h1>
    <nav>
      <!-- Navegación principal -->
    </nav>
  </header>
  
  <main>
    <section>
      <h2>Sección Principal</h2>
      <article>
        <h3>Artículo 1</h3>
        <!-- Contenido del artículo -->
      </article>
    </section>
    
    <aside>
      <h2>Contenido Relacionado</h2>
      <!-- Sidebar o contenido complementario -->
    </aside>
  </main>
  
  <footer>
    <!-- Información del pie de página -->
  </footer>
</body>
</html>

    <section>
      <h2>Noticias Populares</h2>
      <!-- Lista de noticias populares -->
    </section>

      <header>

Buenas prácticas de organización

  • Usa solo un <h1> por página
  • Mantén una jerarquía lógica de encabezados
  • Agrupa contenido relacionado en <section>
  • Usa <article> para contenido que tenga sentido por sí solo
  • Coloca navegación secundaria en <aside>
  • Incluye landmark roles para mayor accesibilidad

SEO básico y accesibilidad

La estructura semántica no solo mejora la organización del código, sino que también es fundamental para el SEO (Search Engine Optimization) y la accesibilidad web.

Beneficios para SEO

  • Mejor comprensión: Los motores de búsqueda entienden mejor el contenido
  • Indexación eficiente: Facilita la indexación de páginas
  • Snippets mejorados: Mejor presentación en resultados de búsqueda
  • Ranking favorable: Google premia la estructura semántica

Atributos importantes para SEO y accesibilidad

Atributo lang

Especifica el idioma del contenido para motores de búsqueda y lectores de pantalla:

<html lang="es">
  <!-- Página en español -->
</html>

<html lang="en">
  <!-- Página en inglés -->
</html>

<p lang="en">This paragraph is in English</p>

Atributo alt en imágenes

Fundamental para accesibilidad y SEO de imágenes:

<!-- Buenas prácticas para alt -->
<img src="producto.jpg" alt="Smartphone Samsung Galaxy S24 color azul">
<img src="logo.png" alt="Logo de TechCorp">
<img src="decoracion.png" alt=""> <!-- Imagen decorativa -->

<!-- Evitar -->
<img src="imagen.jpg" alt="imagen"> <!-- Muy genérico -->
<img src="foto.jpg"> <!-- Sin alt -->

Atributo title

Proporciona información adicional mediante tooltips:

<a href="contacto.html" title="Página de contacto - Envíanos un mensaje">
  Contacto
</a>

<abbr title="HyperText Markup Language">HTML</abbr>

<img src="grafico.png" alt="Ventas 2025" title="Gráfico de ventas enero-octubre 2025">

Meta tags esenciales

Meta tags para SEO:

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- SEO básico -->
  <title>Título específico y descriptivo - Nombre del sitio</title>
  <meta name="description" content="Descripción concisa de la página (150-160 caracteres)">
  <meta name="keywords" content="palabra1, palabra2, palabra3">
  
  <!-- Open Graph para redes sociales -->
  <meta property="og:title" content="Título para redes sociales">
  <meta property="og:description" content="Descripción para redes sociales">
  <meta property="og:image" content="imagen-compartir.jpg">
  <meta property="og:url" content="https://misitioweb.com">
  
  <!-- Twitter Cards -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="Título para Twitter">
  <meta name="twitter:description" content="Descripción para Twitter">
</head>

Elementos adicionales para accesibilidad

Roles ARIA

Proporcionan información adicional a tecnologías asistivas:

<nav role="navigation">...</nav>
<main role="main">...</main>
<aside role="complementary">...</aside>
<div role="button" tabindex="0">Botón personalizado</div>

Atributos ARIA

<!-- Etiquetas descriptivas -->
<input type="email" aria-label="Correo electrónico para newsletter">

<!-- Estados -->
<button aria-expanded="false" aria-controls="menu">Menú</button>
<div id="menu" aria-hidden="true">...</div>

<!-- Descripciones -->
<input type="password" aria-describedby="pwd-help">
<div id="pwd-help">Mínimo 8 caracteres</div>

Estructura de encabezados para SEO

Jerarquía correcta:

<h1>Título principal de la página</h1>
  <h2>Sección importante</h2>
    <h3>Subsección</h3>
    <h3>Otra subsección</h3>
      <h4>Detalle específico</h4>
  <h2>Otra sección importante</h2>
    <h3>Subsección</h3>

Checklist de SEO y accesibilidad

  • ✅ Usar solo un <h1> por página
  • ✅ Jerarquía lógica de encabezados (h1-h6)
  • ✅ Atributo alt en todas las imágenes
  • ✅ Atributo lang en el elemento html
  • ✅ Meta description única para cada página
  • ✅ Títulos descriptivos y únicos
  • ✅ Enlaces con texto descriptivo
  • ✅ Estructura semántica clara
  • ✅ Contraste adecuado en colores
  • ✅ Navegación por teclado funcional

Herramientas útiles

  • Google Lighthouse: Auditoría de performance y SEO
  • WAVE: Evaluación de accesibilidad web
  • aXe DevTools: Extensión para testing de accesibilidad
  • Google Search Console: Monitoreo de SEO
  • Screaming Frog: Análisis técnico de SEO

Ejercicio Práctico

Estructura semántica completa

Intermedio

Objetivo: Divide tu página en secciones con encabezado, menú, contenido principal, y pie de página usando elementos semánticos.

Pasos a seguir:

  1. Crea una estructura HTML5 completa con elementos semánticos
  2. Implementa un header con logo y navegación
  3. Diseña un main con al menos 3 secciones temáticas
  4. Agrega un aside con contenido complementario
  5. Incluye un footer con información de contacto
  6. Optimiza para SEO con meta tags apropiados
  7. Asegura accesibilidad con atributos alt, lang y roles

Estructura sugerida:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>[Tu nombre] - Desarrollador Web</title>
  <meta name="description" content="Portfolio de [tu nombre], desarrollador web especializado en tecnologías modernas">
</head>
<body>
  <header>
    <h1>[Tu Nombre]</h1>
    <nav>
      <ul>
        <li><a href="#sobre-mi">Sobre mí</a></li>
        <li><a href="#proyectos">Proyectos</a></li>
        <li><a href="#contacto">Contacto</a></li>
      </ul>
    </nav>
  </header>
  
  <main>
    <section id="sobre-mi">
      <h2>Sobre mí</h2>
      <!-- Tu presentación -->
    </section>
    
    <section id="proyectos">
      <h2>Mis Proyectos</h2>
      <article>
        <h3>Proyecto 1</h3>
        <!-- Descripción del proyecto -->
      </article>
    </section>
    
    <section id="contacto">
      <h2>Contacto</h2>
      <!-- Información de contacto -->
    </section>
  </main>
  
  <aside>
    <h2>Habilidades</h2>
    <!-- Lista de habilidades -->
  </aside>
  
  <footer>
    <p>© 2025 [Tu nombre]. Todos los derechos reservados.</p>
  </footer>
</body>
</html>

Criterios de evaluación:

  • ✅ Uso correcto de elementos semánticos

  • ✅ Meta tags completos y descriptivos
  • ✅ Atributos de accesibilidad implementados
  • ✅ Navegación funcional y semántica
  • ✅ Estructura clara y organizada
