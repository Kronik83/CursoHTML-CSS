Tablas HTML para organizar datos

Las tablas HTML permiten organizar y mostrar datos de forma estructurada en filas y columnas. Son ideales para presentar información tabular como horarios, precios, estadísticas o cualquier dato que se beneficie de una disposición en rejilla.

Estructura básica de una tabla

Una tabla HTML se compone de varios elementos que trabajan juntos:

  • <table> - Contenedor principal de la tabla
  • <thead> - Encabezado de la tabla
  • <tbody> - Cuerpo principal con datos
  • <tfoot> - Pie de tabla (opcional)
  • <tr> - Fila de tabla
  • <th> - Celda de encabezado
  • <td> - Celda de datos
  • <caption> - Título descriptivo

Tabla básica:

<table>
  <caption>Horarios de Atención</caption>
  <thead>
    <tr>
      <th>Día</th>
      <th>Apertura</th>
      <th>Cierre</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Lunes</td>
      <td>9:00 AM</td>
      <td>6:00 PM</td>
    </tr>
    <tr>
      <td>Martes</td>
      <td>9:00 AM</td>
      <td>6:00 PM</td>
    </tr>
    <tr>
      <td>Miércoles</td>
      <td>9:00 AM</td>
      <td>6:00 PM</td>
    </tr>
  </tbody>
</table>

Tablas responsivas

Para hacer tablas responsivas, se puede usar CSS y técnicas de diseño adaptativo:

Tabla con scroll horizontal:

<div class="table-container">
  <table>
    <!-- Contenido de la tabla -->
  </table>
</div>

<style>
.table-container {
  overflow-x: auto;
}

table {
  min-width: 600px;
  width: 100%;
}
</style>

Buenas prácticas para tablas

  • Usa <caption> para describir el contenido de la tabla
  • Incluye <thead>, <tbody> y <tfoot> cuando sea apropiado
  • Usa scope="col" y scope="row" para mejorar la accesibilidad
  • Evita usar tablas para layout, úsalas solo para datos tabulares
  • Considera la responsividad en dispositivos móviles
  • Mantén un estilo consistente en toda la tabla

Formularios HTML interactivos

Los formularios son elementos fundamentales para la interacción del usuario con una página web. Permiten recopilar información, realizar búsquedas, enviar comentarios y mucho más.

Estructura básica de un formulario

Todo formulario HTML se construye usando estos elementos principales:

  • <form> - Contenedor principal del formulario
  • <label> - Etiqueta descriptiva para campos
  • <input> - Campo de entrada de datos
  • <textarea> - Área de texto multilínea
  • <select> - Lista desplegable
  • <button> - Botón de acción
  • <fieldset> - Agrupación de campos relacionados
  • <legend> - Título para un fieldset

Formulario básico de contacto:

<form action="/enviar-contacto" method="post">
  <fieldset>
    <legend>Información Personal</legend>
    
    <label for="nombre">Nombre completo:</label>
    <input type="text" id="nombre" name="nombre" required>
    
    <label for="email">Correo electrónico:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="telefono">Teléfono:</label>
    <input type="tel" id="telefono" name="telefono">
  </fieldset>
  
  <fieldset>
    <legend>Mensaje</legend>
    
    <label for="asunto">Asunto:</label>
    <select id="asunto" name="asunto" required>
      <option value="">Selecciona un asunto</option>
      <option value="consulta">Consulta general</option>
      <option value="soporte">Soporte técnico</option>
      <option value="ventas">Información de ventas</option>
    </select>
    
    <label for="mensaje">Tu mensaje:</label>
    <textarea id="mensaje" name="mensaje" rows="5" required></textarea>
  </fieldset>
  
  <button type="submit">Enviar mensaje</button>
  <button type="reset">Limpiar formulario</button>
</form>

Atributos importantes del elemento <form>

  • action - URL donde se enviarán los datos
  • method - Método HTTP (GET o POST)
  • enctype - Tipo de codificación para archivos
  • target - Dónde mostrar la respuesta
  • novalidate - Desactiva la validación automática

Diferentes configuraciones de form:

<!-- Formulario para búsqueda (GET) -->
<form action="/buscar" method="get">
  <input type="search" name="q" placeholder="Buscar...">
  <button type="submit">Buscar</button>
</form>

<!-- Formulario para subir archivos -->
<form action="/subir" method="post" enctype="multipart/form-data">
  <label for="archivo">Seleccionar archivo:</label>
  <input type="file" id="archivo" name="archivo" accept=".pdf,.doc,.docx">
  <button type="submit">Subir archivo</button>
</form>

<!-- Formulario que abre en nueva ventana -->
<form action="/procesar" method="post" target="_blank">
  <!-- Campos del formulario -->
</form>

Elemento <label> y accesibilidad

Las etiquetas son cruciales para la accesibilidad y usabilidad:

Formas correctas de asociar labels:

<!-- Método 1: usando el atributo 'for' -->
<label for="usuario">Nombre de usuario:</label>
<input type="text" id="usuario" name="usuario">

<!-- Método 2: envolviendo el input -->
<label>
  Contraseña:
  <input type="password" name="password">
</label>

<!-- Para checkboxes y radios -->
<label>
  <input type="checkbox" name="acepto" value="si">
  Acepto los términos y condiciones
</label>

Validación HTML5

HTML5 incluye validación automática con varios atributos:

Atributos de validación:

<form>
  <!-- Campo obligatorio -->
  <label for="nombre">Nombre (obligatorio):</label>
  <input type="text" id="nombre" name="nombre" required>
  
  <!-- Longitud mínima y máxima -->
  <label for="usuario">Usuario (4-12 caracteres):</label>
  <input type="text" id="usuario" name="usuario" 
         minlength="4" maxlength="12" required>
  
  <!-- Patrón personalizado -->
  <label for="codigo">Código postal (5 dígitos):</label>
  <input type="text" id="codigo" name="codigo" 
         pattern="[0-9]{5}" title="Debe ser exactamente 5 dígitos">
  
  <!-- Rango numérico -->
  <label for="edad">Edad (18-99):</label>
  <input type="number" id="edad" name="edad" 
         min="18" max="99" required>
  
  <!-- Placeholder para ayuda -->
  <label for="telefono">Teléfono:</label>
  <input type="tel" id="telefono" name="telefono" 
         placeholder="+34 123 456 789">
         
  <button type="submit">Enviar</button>
</form>

Mensajes de validación personalizados

Usando JavaScript para personalizar mensajes:

<input type="email" id="email" name="email" required
       oninvalid="this.setCustomValidity('Por favor introduce un email válido')"
       oninput="this.setCustomValidity('')">

<script>
document.getElementById('email').addEventListener('invalid', function(e) {
  if (this.validity.valueMissing) {
    this.setCustomValidity('El email es obligatorio');
  } else if (this.validity.typeMismatch) {
    this.setCustomValidity('Por favor introduce un email válido');
  }
});
</script>

Tipos de inputs en HTML5

HTML5 introdujo muchos tipos nuevos de inputs que mejoran la experiencia del usuario y proporcionan validación automática. Cada tipo está optimizado para diferentes tipos de datos.

Inputs de texto

Tipos básicos de texto:

<!-- Texto simple -->
<label for="texto">Texto normal:</label>
<input type="text" id="texto" name="texto" placeholder="Escribe aquí...">

<!-- Contraseña -->
<label for="password">Contraseña:</label>
<input type="password" id="password" name="password">

<!-- Email (con validación automática) -->
<label for="email">Email:</label>
<input type="email" id="email" name="email" placeholder="usuario@ejemplo.com">

<!-- URL (con validación automática) -->
<label for="website">Sitio web:</label>
<input type="url" id="website" name="website" placeholder="https://ejemplo.com">

<!-- Teléfono -->
<label for="tel">Teléfono:</label>
<input type="tel" id="tel" name="tel" placeholder="+34 123 456 789">

<!-- Búsqueda -->
<label for="busqueda">Buscar:</label>
<input type="search" id="busqueda" name="busqueda" placeholder="Buscar productos...">

Inputs numéricos

Números y rangos:

<!-- Número -->
<label for="edad">Edad:</label>
<input type="number" id="edad" name="edad" min="0" max="120" step="1">

<!-- Rango con slider -->
<label for="volumen">Volumen:</label>
<input type="range" id="volumen" name="volumen" min="0" max="100" value="50">
<output for="volumen">50</output>

<!-- Precio con decimales -->
<label for="precio">Precio:</label>
<input type="number" id="precio" name="precio" min="0" step="0.01" placeholder="0.00">

Inputs de fecha y hora

Fechas y horarios:

<!-- Fecha completa -->
<label for="fecha">Fecha:</label>

<!-- Hora -->

<!-- Fecha y hora combinadas -->

<!-- Mes y año -->

Atributos importantes para inputs

Mejores prácticas para formularios

  • Siempre usa <label> asociado a cada input
  • Agrupa campos relacionados con <fieldset> y <legend>
  • Usa el tipo de input más apropiado para cada dato
  • Incluye placeholder como ayuda, no como sustituto del label
  • Implementa validación tanto en cliente como servidor
  • Proporciona feedback claro sobre errores
  • Usa required solo en campos realmente obligatorios
  • Considera la accesibilidad en todo momento
  • Testa tu formulario en diferentes dispositivos
  • Usa autocomplete para mejorar la experiencia del usuario

Ejercicio Práctico

Tabla de horarios y formulario de contacto

Intermedio

Objetivo: Crear una página que combine una tabla informativa con un formulario funcional de contacto.

Parte 1: Tabla de horarios

  1. Crea una tabla con los horarios de atención de un negocio
  2. Incluye columnas para: Día, Horario de apertura, Horario de cierre, Servicios disponibles
  3. Usa <caption>, <thead>, <tbody> y elementos semánticos
  4. Implementa colspan o rowspan donde sea apropiado
  5. Asegura accesibilidad con scope y otros atributos

Parte 2: Formulario de contacto

  1. Diseña un formulario completo de contacto
  2. Incluye campos para: nombre, email, teléfono, asunto, mensaje
  3. Usa diferentes tipos de inputs apropiados
  4. Implementa validación HTML5
